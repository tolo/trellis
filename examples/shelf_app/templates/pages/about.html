About This Example

Shelf Pipeline

Shelf composes the application as a middleware pipeline around a final handler. This example wraps the router with logging, security headers, Trellis engine injection, CSRF protection, and optional dev-mode live reload.

Request Context

trellisEngine(engine) stores the Trellis engine in the Shelf request context, so handlers can call renderPage() and renderFragment() without globals. Response helpers also merge request-scoped values like csrfToken into the template context automatically.

Middleware Ordering

trellisSecurityHeaders() runs outside the rest of the app so every response gets the same policy. The Trellis engine middleware must run before trellisCsrf() so CSRF-aware handlers can still resolve the engine from the request context.

CSRF Protection

trellisCsrf() uses the double-submit cookie pattern. Safe requests mint a signed token cookie, and HTMX sends that token back on mutations via the X-CSRF-Token header. Hidden form fields remain as a fallback for non-HTMX requests.

Dev Hot Reload

When you run the example with --dev or set DEV=true, devMiddleware() watches the template directory and injects a small SSE client into HTML responses. Template edits refresh in the browser without restarting the server.
